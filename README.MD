Данный проект содержит серию автотестов для проверки API учебного сервиса Яндекс.Самокат.
https://qa-scooter.praktikum-services.ru/

Его документация: https://qa-scooter.praktikum-services.ru/docs/

Проект создан с помощью Java11, JUnit4, RestAssured и Allure

_**Описание классов**_

BaseTestApi содержит общие действия для всех тестовых классов: 
- Задание базового адреса для обращения к API сервиса с помощью protected static final String BASEURL = "https://qa-scooter.praktikum-services.ru"; и .setBaseUri(BASEURL);
- Параметр application/json сообщает серверу, что тело запроса содержит данные в формате JSON.
- Параметр charset=UTF-8 указывает кодировку символов (UTF‑8), чтобы сервер корректно интерпретировал текст (в т.ч. кириллицу).

CourierTestData служит моделью данных (DTO) для передачи информации о курьере при выполнении API‑тестов CreateCourierTest и CreateCourierWithRandomNameAuthAndDeleteTest. 
Этот класс:
- Хранит данные. Приватные поля (login, password, firstName) инкапсулируют данные, которые будут отправлены в теле запроса.

- Осуществляет инициализацию. Конструктор с параметрами позволяет создать объект с заранее заданными значениями — например, для тестирования создания курьера с конкретными учётными данными.

- Производит сериализацию в JSON. Геттеры обеспечивают доступ к полям для библиотек сериализации (например, Jackson), которые автоматически преобразуют объект в JSON‑формат для передачи в API.

GetOrdersTestData по аналогии с CourierTestData передаёт информацию о заказе и используется в тестовом классе GetOrdersTest

OrdersTestData используется для параметризованного теста OrderTest



**_Описание тестовых классов_**

AuthCourierTest используется для проверки регистрации ранее зарегистрированного пользователя. Успешный запрос возвращает id пользователя.
Также этот класс можно использовать для проведения негативных проверок:
- Если не передать одно из полей, система возвращает ошибку 400 и сообщение "Недостаточно данных для входа"
- Если передать неправильные логин и/или пароль, система возвращает ошибку 404 и сообщение "Учетная запись не найдена"

CreateCourierTest используется для проверки создания нового пользователя. Успешный запрос возвращает `ok: true`
Также класс используется для проведения следующих проверок:
- Нельзя создать двух одинаковых курьеров (Система возвращает ошибку 409 и сообщение "Этот логин уже используется. Попробуйте другой.")
- Чтобы создать курьера, нужно передать все обязательные поля (При отсутствии логина и/или пароля система возвращает ошибку 400 и сообщение "Недостаточно данных для создания учетной записи")

CreateCourierWithRandomNameAuthAndDeleteTest создаёт пользователя со случайным логином при помощи currentTimeMillis(), авторизует пользователя, получает id и удаляет пользователя

GetOrdersTest используется для получения списка заказов курьера по id курьера. Здесь есть баг - система возвращает ответ не в ожидаемом формате

OrderTest - параметризованный тест для проверки создания заказа. Содержит три набора данных:
- Первый делает заказ с цветом BLACK;
- Второй делает заказ с цветом GREY;
- В третьем заказе цвет не указан
Тело ответа содержит `track`
